# Japanese Drill App — Project Summary

## What this is
A single-file HTML/CSS/JS Japanese learning app (`index.html`), hosted on GitHub Pages, built and used by Andrew and ~9 classmates for an in-person Japanese course, ahead of a Japan trip. No build step, no framework — deliberately kept as one file for simplicity (classmates just open it and it works). Currently ~3,530 lines.

**Working style established across many sessions:** draft/direct, Claude builds and validates. Every change gets syntax-checked, structurally validated, and functionally tested with real code before being presented as done — not just visually inspected. Claude asks before making judgment calls on ambiguous design decisions rather than assuming. Dead code/data gets cleaned up when found, not left to accumulate. When something is "roughly right but not quite," the instinct has consistently been to find and fix the actual root cause rather than patch the symptom.

## Architecture
- **Kana**: hiragana/katakana, each with a "selection screen" (row-based selection by gojuon section, confidence-shaded tiles, drill button)
- **Kanji**: 8 thematic sections (numbers-time, nature-body [now includes weather], directions-travel, food, places, people, general, verbs), each with its own tile-selection screen. Kanji always tests *meaning* (never sound) — options now show English prominently, romaji small/bracketed (deliberately different from vocab, which tests either).
- **Vocab**: 13 categories across Level 1, Level 2, and an "Extra" tier (Colours, Trip-prep). Every category has its own tile-selection screen (browse, individually select words, drill, scored). A "drill by meaning vs sound" toggle exists per-category with independent state (not shared/inherited across categories).
- **Read page**: numbers/times (one-tap, scored with red/green swatches), separated by a gold divider from a "random pool" section below (reading words L1/L2, reading words everything, random kanji) — those three are select-then-start with a shared toggle and Start button.
- **Scoring**: every drillable thing (kana, kanji, vocab, numbers, times) has a confidence score from -2 to +2, ±1 per answer for most things, ±0.1 per answer for numbers/times (aggregate, not per-item). Colours' tiles additionally get a border that shows hue+intensity matching the tile's own colour, on top of the standard red/green background shading everyone else uses.

## Terminology (settled, worth using consistently)
- **"Selection screen"** = interactive tile+score+drill screens (most things)
- **"Reference"** = reserved only for two genuinely pure-browse screens: the ordered Days-of-week list and the Minute-endings pattern table
- Code was fully renamed months ago to match this (function names, screen ids all say "Selection" not "Reference" where appropriate)

- **Why Vocab and Kanji are organized so differently**: Vocab's categories come directly from the actual lessons taught in the in-person course — its structure mirrors how the curriculum itself grouped things (e.g. a dedicated Verbs lesson/category exists because the course taught verbs together). Kanji started from a completely different brief — "give me a set of kanji useful for travel, my trip, and survival," which then expanded into "give me N5" (full JLPT N5 coverage) — and its thematic sections (Travel, Food, etc.) were a categorization layer applied afterward, not inherited from a lesson plan. This is why kanji verbs ended up scattered thematically (open/close/push/pull in Travel, eat/drink in Food, only the leftover general-purpose ones in Verbs) while vocab verbs didn't — it's two different provenances behaving sensibly for their own origin, not an inconsistency needing a fix.
